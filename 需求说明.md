# FDE管理平台 · 需求说明

## 1. 项目背景

团队原有 **FDE 审批式工单流程** 偏重过程合规，需求录入、人员分配、进度跟踪、验收均依赖手工表单，效率低、信息不透明。

本项目目标是建设 **FDE管理平台**：以敏捷管理方法为核心，结合国产大模型（DeepSeek + 阿里云 Paraformer），实现从需求输入到任务分配、执行跟踪、验收交付的全链路数字化与智能化。

- **本地地址**：http://localhost:3456
- **代码仓库**：https://github.com/Rabit11/FDE-
- **开发方案**：public/FDE管理平台开发方案.html

---

## 2. 目标用户

共 **12 名** 预置用户，按职能分为两类：

| 类型 | 人员 | 能力 |
|------|------|------|
| 管理员 | admin、曾锐、张弛、林宇飞、毛研勋 | 审核 + 提出需求 + 执行 + 人员管理 |
| 普通成员 | 赵立泽、王诗瑶、万贤书、刘紫薇、王紫薇、杨会冉、杨子豪 | 提出需求 + 执行 |

登录方式：**工号 + 密码**（成员默认密码与工号相同）。

## 3. 核心需求

### 3.1 多职能角色与工作台

每人可兼任多个职能，平台按能力动态生成导航与工作台：

| 职能 | 主要功能 |
|------|----------|
| 审核人员 | 审核分配、Sprint 管理、验收中心、团队看板 |
| 需求提出人员 | 文本/语音/文档提交需求，AI 自动拆分任务 |
| 执行人员 | 我的工作台、流动看板、进度更新 |
| 个人主页 | 维护并查看历史项目、技能、当前任务（全员可互访） |
| 人员管理 | 团队成员查看与管理（仅管理员） |

### 3.2 个人主页

每位成员可填写：

- 个人简介、技能标签
- **历史做过的项目**（项目名、角色、描述）
- 可用状态（空闲 / 有限 / 繁忙）
- 系统自动展示手上任务与协助任务

### 3.3 AI 智能人员分配

审核人员在「审核分配」页，平台根据以下信息自动推荐人选：

- 任务需求描述
- 执行人员项目背景与技能匹配度
- 当前工作负载（WIP、Story Points）

输出结果：**1 名主执行人 + 1~N 名协助执行人**，并附推荐理由。支持算法引擎 + DeepSeek LLM 双引擎。

### 3.4 敏捷管理

| 模块 | 需求 |
|------|------|
| 流动看板 | 六列 Kanban，拖拽流转，WIP 上限告警 |
| Sprint | 2 周迭代，燃尽图，Sprint Backlog |
| Backlog | Epic / Story / Task / Bug 层级管理 |
| 验收中心 | 在线验收，打回后自动创建返工 Story |
| 度量仪表盘 | Lead Time、吞吐量、活动流 |

### 3.5 AI 能力（国产模型）

| 环节 | 模型 | 能力 |
|------|------|------|
| 语音转文字 | 阿里云 Paraformer | 会议录音 → 中文转写 |
| 需求分析 | DeepSeek V3 | 梳理 Markdown 需求文档 |
| 任务拆解 | DeepSeek V3 | Epic → Story → Task + 验收标准 |
| 智能分配 | DeepSeek V3 | 主执行人 + 协助人推荐 |
| AI 协作者 | DeepSeek V3 | 对话式项目管理助手 |
| 站会/风险/Retro | DeepSeek V3 | 自动生成洞察报告 |

**语音/文档 → 任务流水线**：

```
上传语音/录音 → 转写 → AI 梳理文档 → 拆分任务 → 智能分配 → 写入 Backlog
```
