1. 背景与目标

核心判断：v3.0 全局看板已具备四态指标、人员活跃、甘特、迷你看板、AI 洞察等完整能力，但模块纵向堆叠达 7 层，领导需滚动才能看完，无法在 10 秒内回答「要不要介入」。本方案在保留全部后端能力前提下，重构为「驾驶舱」信息层级。

优化目标（三问）

问题目标时间
整体健康吗？流动健康度 + 四态 RAG 指标卡3 秒
哪里红了？行动队列：阻塞 / P1 / 超期 / 无进展5 秒
我今天该处理什么？一键跳转任务中心对应筛选10 秒

2. 现状诊断（v3.0）

当前页面结构（自上而下 7 层，需大量滚动） ① 流程步骤条（装饰性） ② 四态指标卡 ×4 + KPI 胶囊条 ×4 ③ 三栏：人员活跃 / 任务态势 / 实时动态 ④ 甘特图（全宽，占屏大） ⑤ 四态迷你看板 ⑥ 最近动态（8 条） ← 与 ③ 重复 ⑦ AI 洞察（3 条） ← 与顶栏站会/风险扫描脱节
问题表现影响
信息过载可见 KPI 约 12+，超 Miller 法则 7±2扫描成本高，决策延迟
内容重复「实时动态」出现 2 次浪费首屏，降低信任
缺行动焦点阻塞仅显示数字「1」看到问题不知下一步
KPI 无上下文吞吐量/Lead Time 无趋势、无目标数字不可决策
模块重叠人员活跃 vs 任务态势认知负担重复
甘特过重与迷你看板功能重叠分析型内容挤占首屏
未打通 P1v3.0 三级优先级未在驾驶舱体现战略与执行脱节

3. 设计原则

倒金字塔布局

参考 Executive Dashboard Best Practices：头条 KPI → 趋势洞察 → 行动明细。

颜色只表状态

RAG（绿/黄/红）仅用于 WIP 超限、阻塞、P1 告警；装饰性渐变减至流程条与顶栏。

渐进披露

甘特图默认折叠，与四态快照 Tab 切换；详情点击下钻任务中心，不在首屏堆满图表。

与业务闭环

行动队列规则与「今日工作台」筛选（待办/阻塞/今日未报进展）使用同一套数据逻辑。

4. 信息架构（v4.0 三层驾驶舱）

L1 态势层（首屏必见） ├─ 流动健康度 RAG 徽章 + 刷新条 ├─ 四态指标卡（可点击 → 任务中心筛选） └─ KPI 胶囊：吞吐 / Lead Time / P1 / 今日无进展 / 超期 L2 洞察层（次屏） ├─ 左：需立即处理 · 行动队列（阻塞→P1→超期） └─ 右：近 7 日流动趋势（堆叠面积/折线） L3 明细层（可折叠） ├─ 左：人员负载（合并活跃+态势） ├─ 右：四态快照 / 甘特图 [Tab] └─ 底：AI 洞察横幅 + 合并动态流（去重）

7. KPI 体系

保留并增强

  • 四态数量 + RAG 边框 + 7日 sparkline
  • 近14天吞吐量、平均 Lead Time（加趋势箭头）
  • WIP 使用率（超限红色强调）

新增（与 v3.0 优先级打通）

  • P1 待推进：priority=1 且进行中
  • 今日无进展：与工作站逻辑一致
  • 超期任务：ganttEndDate < now
  • 流动健康度：综合分 0–100

RAG 阈值建议

指标🟢 绿🟡 黄🔴 红
执行中 WIP≤ 80% 上限80%–100%> 100%（当前 200%）
阻塞数0—≥ 1
流动健康度≥ 8060–79< 60
P1 今日无进展01–2≥ 3

8. 交互与性能

  • 去重：删除底部「最近动态」，仅保留合并动态流（最多 8 条，按时间倒序）
  • 刷新：页面可见 15s 轮询，隐藏时 60s；优先 PATCH 数字 DOM，避免全量 innerHTML 闪烁
  • 下钻：所有 KPI、行动项、人员行、快照卡片均可点击，参数与任务中心筛选对齐
  • 顶栏联动：站会摘要 / 风险扫描完成后，L3 洞察横幅置顶展示 high severity 结果

9. 实施分期

阶段内容工期收益
