2. 四层架构详解

L1

感知层 Perception — 多模态输入理解

语音（Paraformer-v2 主引擎 + SenseVoice 降级）、文档（PDF/Word/TXT/MD 多遍解析）、纯文本会议纪要。输出统一 Markdown 会议记录，经敏捷热词校正提升领域准确率。

L2

认知层 Cognition — 大模型推理与结构化

DeepSeek-Chat 承担日常拆分/对话；DeepSeek-Reasoner 处理复杂规划；通义千问 Qwen-Max 作为国产备选。强制 JSON 输出驱动下游 API，温度参数按场景调优（拆分 0.3 / 对话 0.5 / 创意 0.7）。

L3

决策层 Decision — RAG + Agent + 算法

实时注入看板数据、团队负载、阻塞项、个人技能画像；Intent 路由 8 类意图；智能分配采用「技能匹配 + 负载均衡 + 可用性」三维评分，LLM 增强推荐理由。

L4

执行层 Action — 业务闭环

AI 拆解结果一键填入需求表单；自动审核备案并派单；执行人每日进展写入时间线；领导四态操作（终止/二次分配/退回）与 AI 洞察联动。

4. 多模态管线

4.1 会议/语音 → 结构化需求

🎙️ 录音/上传→ ASR 转写→ 热词校正→ 会议记录存储→ LLM 拆解→ 填入需求表单→ 自动派单

语音通路 meeting.js → asr.js

  • 支持 webm / mp3 / wav / m4a 等格式
  • 主引擎 paraformer-v2，失败自动切换 sensevoice-v1
  • 敏捷热词表：Story、Epic、Kanban、WIP、站会、阻塞…
  • 转写结果存为 meeting_record 文档

文档通路 meeting.js → docparse.js

  • 支持 txt / md / csv / pdf / docx
  • 多遍解析提取章节与元数据
  • 统一输出 Markdown 会议记录格式
  • 与语音记录共用「AI 拆解并填入表单」入口

4.2 LLM 会议解析输出 Schema

{ "title": "需求标题", "scene": "业务场景描述", "pain_points": ["痛点1", "痛点2"], "acceptance_criteria": "Given/When/Then 验收标准", "priority": "high|medium|low", "estimated_sp": 5, "suggested_stories": [ { "title": "用户故事", "sp": 3, "criteria": "..." } ] }

5. Agent 框架（RAG + Intent + Memory）

FDE Agent 不是简单 ChatBot，而是绑定实时项目状态的敏捷教练。每次对话自动构建 RAG 上下文包，经 Intent 路由后注入场景化 System Prompt，保留最近 12 轮对话记忆。

5.1 RAG 上下文包结构

buildRagContext(user) → { current_user: { name, role, capabilities }, flow: { in_progress, blocked, archived, terminated }, metrics: { throughput, avgLeadTime, blocked }, my_tasks: [ 最多15项个人任务 ], blocked_items: [ 阻塞详情 ], pending_review: [ 待验收 ], pending_assign: [ 待分配 ], team: [ 每人 workload / skills / availability ], recent_changes: [ 最近10条活动日志 ] }

5.2 Intent 意图路由

Intent触发词示例Agent 行为
split拆分、Epic、Story、需求基于 INVEST 原则给出 Epic→Story→Task 建议
assign分配、指派、谁来做结合团队负载与技能推荐主执行+协助人
risk风险、阻塞、延期分析阻塞项，给出 24h 可执行行动
standup站会、今日、昨日进展生成站会三段式摘要
mywork我的任务、待办按优先级排列个人 Focus List
team团队、负载、谁最忙负载均衡分析与调配建议
acceptance验收、归档待验收任务优先级排序
general其他综合项目数据给出敏捷管理建议

5.3 深度推理模式

当用户启用深度分析（options.deep = true）且配置了 DeepSeek Key 时，自动切换至 deepseek-reasoner 模型，适用于复杂规划、多任务依赖分析、Retro 回顾等场景。

变量必需作用
DEEPSEEK_API_KEY推荐启用 LLM 智能拆分、Agent、站会/风险
DASHSCOPE_API_KEY推荐启用语音转写；同时作为 Qwen LLM 备选
DEEPSEEK_REASONER_MODEL可选Agent 深度推理模式
ASR_MODEL可选指定 ASR 主引擎，默认 paraformer-v2

10. 业界方案对比

传统项目管理工具

  • Jira / 禅道：无原生 LLM
  • 依赖人工拆分与分配
  • 语音/会议需第三方
  • AI 能力需插件拼接

通用 AI 助手

  • ChatGPT / Copilot：无项目上下文
  • 不绑定看板四态流转
  • 输出非结构化，难落地
  • 无国产模型合规路径

✅ FDE AIMF 框架

  • RAG 注入实时项目数据
  • 多模态 → 结构化 → 自动派单
  • DeepSeek + DashScope 国产双引擎
  • LLM + 规则双引擎永不宕机
  • Intent Agent 场景化精准回复
