0. 详细前言：FDE 到底用的是什么「模型」？

核心定义

FDE管理平台 不内置、不下载、不部署任何大模型权重文件（无 .gguf / .safetensors / ONNX）。所有「智能」能力通过 HTTP API 调用云端模型实现；当 API 不可用时，自动降级到 本地规则引擎（纯 JavaScript 逻辑，见 server/ai/engine.js、assign.js）。

0.1 三类能力来源

类型是否需要 API Key是否需要下载模型本项目实现位置
云端 LLM 大语言模型 是（推荐 DeepSeek） 否 — 调用远程 API server/ai/llm.js
云端 ASR 语音转写 是（DashScope 同一 Key） 否 — 上传音频到云端转写 server/ai/asr.js
本地解析 文档/规则 否 仅 npm 包（pdf-parse、mammoth） server/ai/docparse.js、engine.js

0.2 你需要准备什么？

  1. DeepSeek API Key（认知层主模型）— 用于需求拆分、Agent 对话、站会/风险、会议解析
  2. 阿里云 DashScope API Key（感知层 + LLM 备选）— 用于语音转写 Paraformer；无 DeepSeek 时兼作 Qwen LLM
  3. 项目根目录 .env 文件 — 从 .env.example 复制后填入 Key
  4. 不需要：GPU 服务器、模型文件、Docker 推理镜像、HuggingFace 下载

0.4 数据流技术路径（从源码归纳）

用户输入（文本/语音/文档）
    │
    ├─ 文本 ──────────────────► llm.chat() ──► DeepSeek/Qwen API
    │
    ├─ 语音 ──► asr.transcribeFile() ──► DashScope ASR API
    │              │ 级联: paraformer-v2 → sensevoice-v1 → qwen-audio-turbo
    │              └─► 转写文本 ──► llm.parseMeetingToRequirement()
    │
    ├─ 文档 ──► docparse.parseDocument() [本地]
    │              └─► llm.analyzeDocument() [多遍分块 + 合成]
    │
    └─ Agent ──► agent.buildRagContext() [本地 DB 注入]
                   └─► agent.agentChat() ──► llm.chat() [可选 deepseek-reasoner]

3. 供应商路由优先级（源码逻辑）

摘自 server/ai/llm.js → getActiveProvider()：

function getActiveProvider() {
  if (process.env.DEEPSEEK_API_KEY) return 'deepseek';      // 优先级 1
  if (process.env.DASHSCOPE_API_KEY) return 'dashscope';    // 优先级 2（兼作 LLM）
  if (process.env.OPENAI_API_KEY) return 'openai';          // 优先级 3
  return null;  // → 全部 AI 功能降级到本地规则引擎
}

关键影响

  • 同时配置 DeepSeek + DashScope 时：LLM 走 DeepSeek，ASR 仍走 DashScope（ASR 只认 DASHSCOPE_API_KEY）
  • 只配 DashScope：LLM 使用 qwen-max，ASR 使用 paraformer-v2
  • 都不配：平台正常运行，登录页显示「规则模式 + LLM 可选增强」

4.1 注册与获取 API Key 步骤

1

打开 platform.deepseek.com，使用手机号或邮箱注册账号

2

进入控制台 → API Keys → 点击「创建 API Key」

3

复制以 sk- 开头的 Key（只显示一次，请立即保存）

4

充值余额（按 Token 计费，新用户通常有免费额度，以官网为准）

5

写入项目 .env：DEEPSEEK_API_KEY=sk-你的密钥

6

重启服务 npm start，控制台应显示：大模型: DeepSeek (deepseek-chat)

4.2 本项目实际请求体（摘自 llm.js）

POST https://api.deepseek.com/v1/chat/completions
Authorization: Bearer ${DEEPSEEK_API_KEY}
Content-Type: application/json

{
  "model": "deepseek-chat",           // 或 deepseek-reasoner
  "messages": [
    { "role": "system", "content": "..." },

12. 计费参考（请以官网实时价格为准）

服务计费单位参考策略官网价格页
DeepSeek Chat 输入/输出 Token 拆分一次约 2K~8K tokens；Agent 对话按轮次累计 DeepSeek 定价
DeepSeek Reasoner Token（含思维链） 仅深度分析时触发，成本高于 chat 同上
Paraformer ASR 音频时长（秒/分钟） 会议录音按分钟计费；短语音成本极低 DashScope 计费
Qwen Max Token 仅作 LLM 备选时使用 同上

💡 团队日常使用建议：DeepSeek 负责所有文本智能（性价比高），DashScope 仅用于语音转写，可将成本控制在每月数十元至数百元量级（视使用量而定）。
