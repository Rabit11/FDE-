3. 图表说明

可视化数据来源交互
环形饼图 flowCategory() 四态计数 点击扇区/图例 → goTaskCenter(flow)
今日更新率环 执行中任务 ∩ 今日 progress_updates 悬停显示已报/未报名单
人员活跃热力条 7 日进展次数 + 主执行负载 点击 → 按人筛选任务中心
任务脉搏（7 柱） 每项任务过去 7 天每日是否有进展 最后一柱绿色=今日；点击行 → 任务详情

任务脉搏图例

每根小柱代表一天（左→右 = 6天前→今天）。蓝色=当日有进展，绿色=今天且已报，灰色=无进展。比纯文字「今日未报 9」更直观看到哪几个任务连续多天没动。

4. 与现有模块关系

保留

  • 流动健康度评分
  • 精简 KPI 条（吞吐量 / Lead Time / P1 / 超期）
  • 行动队列、7 日趋势
  • 四态快照/甘特（全宽）

本方案新增

  • 四态环形饼图（分布一眼看清）
  • 执行中专属洞察面板
  • 今日更新率环（RAG）
  • 任务脉搏 7 日热力

5. 落地建议

步骤改动复杂度
1新增 buildFlowDistribution() + CSS 饼图 / SVG 环低
2新增 buildExecutingInsight()：今日率、人员热力、任务脉搏中
3插入 renderGlobalKanbanBody() 指标卡下方新一行低
4样式 .gk-viz-* 写入 style.css低

纯 CSS 饼图即可满足需求，无需引入 Chart.js；若后续要交互悬停扇区，可换 SVG + stroke-dasharray。
